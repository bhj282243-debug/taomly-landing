FAQ

Common questions.

What does "white label" mean exactly?
+
You (or your client) gets the platform running under a completely custom brand. The Telegram bot has a custom name, the web app shows the restaurant's logo and colors, and the URL is a custom domain. End customers never see "Taomly" — they see the restaurant's brand.
How long does it take to deploy a new restaurant?
+
Once the platform is deployed, adding a new restaurant takes 2–3 minutes in the agency admin panel: enter the restaurant name, slug, Telegram bot token, and brand colors. The system auto-generates webhooks, PWA manifest, and QR codes. No developer involvement needed.
Does each restaurant need a separate Telegram bot?
+
Yes — each restaurant gets its own dedicated Telegram bot (@YourRestaurant_bot). This is what enables full white labeling. Creating a bot via @BotFather takes 30 seconds, and the platform handles all webhook configuration automatically.
Can I buy the source code and host it myself?
+
Yes. The platform is available as an asset sale — full source code, documentation, deployment guides, test suite and architecture docs. Deploy on any cloud provider that supports Python/Docker. Contact us to discuss licensing terms.
What database and infrastructure does it use?
+
PostgreSQL (tested on Neon, works on any Postgres). Backend is FastAPI + SQLAlchemy. Deployable via Docker on Render, Railway, DigitalOcean, AWS, GCP. Includes docker-compose.yml, Dockerfile, Alembic migrations and GitHub Actions CI/CD pipeline.
